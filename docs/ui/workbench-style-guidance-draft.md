# AiBrain 智脑工作台风格指导（暂定）

本文用于后续改造各个智脑工作台时参考。目标不是把所有工作台做成一模一样，而是统一“工作台产品感”和交互骨架，同时保留每个专家/智脑自己的气质、任务特点和领域语言。

## 1. 总体原则

1. 工作台优先是“可用工具”，不是宣传页、人物卡页或聊天皮肤。
2. 首屏应直接进入可操作状态：左侧任务区、中间主功能区、右侧聊天区尽量同时可见。
3. 每个智脑都要有自己的领域风格，但不要牺牲清晰、克制、专业。
4. 页面文案要短，按钮和入口要明确；不要把说明文字堆成介绍页。
5. 中间主功能区承载结构化功能，右侧聊天区承载自由沟通，两者不要互相污染。

## 2. 标准布局

推荐基础结构：

- 顶部栏：返回、首页、工作台名称、副标题、积分入口。
- 左侧任务区：稳定展示该智脑的主要工作流入口。
- 中间主功能区：展示左侧任务对应的表单、工具、卡片、课程、测试、报告等。
- 右侧聊天区：保留用户与智脑的自由对话记录。

布局参考：

- 左侧任务区宽度固定，适合快速切换。
- 中间主功能区尽量宽，承载核心操作。
- 右侧聊天区固定宽度，保持输入和历史记录稳定。
- 顶部栏左边界要和下方主内容左边界对齐。

## 3. 主功能区与聊天区的关系

左侧任务点击后，如果结果展示在中间主功能区：

- 右侧聊天区不要重复显示这次任务的 prompt 和 AI 回复。
- 右侧聊天区应保持“自由沟通记录”的完整和干净。
- 中间主功能区要负责展示该任务的结构化结果。
- Markdown 内容要渲染成排版后的文本，不要裸露 `#`、`####`、`**` 等符号。

右侧输入框手动发送的内容：

- 必须进入右侧聊天记录。
- 可以根据任务需要影响中间区，但不能破坏右侧记录完整性。

## 4. 人物设定区

涉及人物设定、专家身份、使用边界、工作方式的内容，不要做成大面积介绍页。

推荐做法：

- 放在中间主功能区顶部。
- 尽可能与当前任务操作区并排或紧凑排列。
- 人物设定区固定在顶部或作为顶部信息带存在。
- 中间主内容滚动时，不要影响人物设定区的位置。
- 人物设定只保留对用户决策有用的信息，不暴露内部 prompt、蒸馏标签或来源痕迹。

## 5. 配色方向

统一要求：

- 配色要服务领域气质。
- 不要所有工作台都用同一套绿色、棕色或灰色。
- 色彩要克制，不要做成强营销页面。
- 交互状态要清楚：选中、悬停、禁用、错误、成功都要能区分。

示例方向：

- 一修小和尚：偏禅意、安静、留白，可用竹青、米白、墨灰、淡金等。
- 松月先生：偏中国传统审美，可用黛蓝、朱砂、宣纸白、松烟墨等。
- 鸣锋顾问：偏商务、法律/咨询工具感，可用深灰、海军蓝、冷白、低饱和金属色等。
- 松柏先生：偏养生、中医学习工作台，可用松木色、药柜棕、草本绿、暖白。
- 青禾写作类：偏教学、批改、资料交付，可用纸张白、批注红、青绿、浅灰。

## 6. 左侧任务区

左侧任务区要像工具目录，不像广告导航。

要求：

- 任务名称短而明确。
- 每个任务下面可以有一句短提示。
- 当前选中项要明显。
- 分组可用细线区隔。
- 尚未开放的功能可以保留入口，但点击后统一显示“即将上线”，不要进入半成品流程。

## 7. 中间主功能区

中间区是核心生产力区域。

要求：

- 每个任务都有明确的默认空状态。
- 表单项少而关键，避免一上来就让用户填长表。
- 结果展示应结构化：卡片、表格、步骤、报告、测试反馈等。
- 如果内容较长，只让主内容滚动，不影响顶部设定区和整体布局。
- 对于测试、课程、工具类功能，可以增加趣味性，但要围绕学习或产出效率。

## 8. 右侧聊天区

右侧聊天区用于自由补充和连续沟通。

要求：

- 输入框始终可见。
- 聊天历史保持完整。
- 不要把左侧任务自动触发的结构化结果重复塞进聊天区。
- 附件、复制、下载等操作保持一致。
- 输入框 placeholder 可以体现该智脑领域，但不要堆说明。

## 9. 趣味性原则

趣味性不是装饰，而是让功能更愿意被使用。

可以使用：

- 闯关感、进度条、连击、称号、等级。
- 领域化隐喻：药柜、案台、讲义、卷宗、诗笺、禅房等。
- 更有现场感的题面、结果反馈、空状态文案。
- 小而清楚的交互反馈。

避免：

- 大面积装饰背景。
- 与领域无关的卡通化。
- 过度口号化、营销化。
- 把功能藏在漂亮但不清楚的界面里。

## 10. 即将上线功能

尚未稳定的能力不要做半成品。

处理方式：

- 左侧可以保留入口。
- 点击后中间主功能区显示“即将上线”。
- 简短说明未来会支持什么。
- 不显示不可用的上传、提交、生成按钮。
- 不触发真实接口，避免用户遇到失败提示。

## 11. 改造检查清单

每改一个工作台，至少检查：

- 顶部栏和主内容左边界是否对齐。
- 左侧任务区是否清晰，选中态是否明显。
- 中间主功能区是否承担主要任务。
- 右侧聊天区是否只保留自由沟通记录。
- Markdown 是否正确渲染。
- 人物设定是否紧凑、稳定、不随正文滚动错位。
- 配色是否符合该智脑气质。
- 空状态、加载态、错误态、禁用态是否完整。
- 未上线功能是否统一显示“即将上线”。
- 移动端或窄屏下文字是否溢出、按钮是否挤压。

## 12. 当前参考样式

松柏先生养生工作台作为当前参考样式：

- 左侧任务区：任务切换、分组、试验区入口。
- 中间主功能区：每日新知、经方解读、跟我学、中医闪考、菜谱定制、体质辨识等结构化功能。
- 右侧聊天区：保留自由中医咨询沟通。
- 未上线能力：舌头/指纹/面容拍摄辨识统一显示“即将上线”。

后续其他工作台应参考其“骨架”和“清晰度”，而不是直接复制其颜色和中医语汇。
