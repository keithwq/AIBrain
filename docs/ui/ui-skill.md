# AIBrain 专家工作台 UI Skill

## 适用范围

本规范用于改造 AIBrain 项目内的专家工作台，尤其是从“通用聊天页”升级为“专家方法驱动的工作台”时使用。

青禾写作教学工作台是当前标准样板。后续改造其他专家时，先复用这里的信息架构、留白策略、材料库机制和输出交付逻辑，再根据专家领域调整任务树和文案。

## 核心判断

专家工作台不是表单堆叠，也不是普通聊天窗口换皮。

一个合格工作台必须回答三件事：

1. 用户现在要完成什么任务。
2. 专家方法如何参与判断。
3. 最后交付什么可用成品。

对于青禾工作台，关键不是“批改、讲义、说课稿这些机械活交给 AI”，而是这些看似机械的教师交付物都必须先经过青禾写作工作台的教学判断，再生成成品。

## 标准信息架构

采用三栏结构：

左侧：任务树  
用于选择工作类型。只放任务，不放大量说明。任务树标题下直接显示当前任务内容，避免另起一张“当前任务”卡。

中间：工作区  
这是主区域。默认保持空白，用低存在感灰色提示承接操作。开始生成后，这里显示青禾工作台给出的成品和意见。不要在空状态铺满功能说明、方法卡、宣传语。

右侧：材料库  
用于放作文、题目、教案、讲义、说课稿、素材包、补充要求。材料库必须可隐藏，因为不同业务场景下不一定需要一直占屏。

底部：固定输入栏  
输入栏与主按钮并排固定在主工作区底部。上方内容溢出时滚动，页面不跳动。

## 视觉标准

整体采用温润浅色，不做强烈科技感、冷玻璃感、大渐变或营销页视觉。

推荐基调：

- 背景：暖白、浅米、浅灰
- 主文字：石墨黑或深褐黑
- 辅助文字：灰色
- 强按钮：深褐黑
- 边框：浅米灰
- 卡片：白色或轻微暖白

字体和密度：

- 工作台字体整体偏小，避免大标题压迫。
- 左右栏标题使用中等字重，不用大黑粗。
- 中间空状态使用灰色提示，不用彩色宣传卡。
- 边距克制，卡片内距不要过大。
- 任务标题允许换行，不能横向撑出容器。

## 交互原则

1. 控件少，问答补细节  
不要用大量控件覆盖所有可能性。必要字段可选，复杂要求交给对话补充。

2. 任务先行  
用户先选任务树：批改评讲、写作教学、资料诊改、课后沟通。不要让用户先理解系统结构。

3. 材料优先  
允许用户上传自己的资料，让专家指出不足、修改、升格。上传材料不是附件摆设，必须进入模型上下文。

4. 输出可下载  
AI 生成答案后必须提供下载 Word 按钮，减少复制粘贴。

5. 空状态留白  
默认工作区不展示满屏内容，只提示下一步。工作台要像可以开始工作的桌面，不像说明书。

## 青禾工作流基准

青禾工作台至少包含：

- 批改评讲：作文快批、讲评课、课后沟通
- 写作教学：整理教案、素材讲义说课、资料诊改

输出必须体现青禾写作方法：

- 题干限制
- 种籽句
- 主钉子
- 下一刀
- 动作型任务
- 文学 / 应试双轨

资料诊改要先指出不足，再给修改意见和可用改稿。不能只做语言润色。

## 禁忌

- 不要把专家工作台做成 landing page。
- 不要把工作台做成控件大全。
- 不要把中间工作区塞满说明卡。
- 不要把机械提效和专家判断割裂开。
- 不要默认所有作文教学都是一二三稿，轮次要可扩展。
- 不要冒充官方阅卷标准。
- 不要替学生生成整篇可提交作文。
