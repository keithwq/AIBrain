# 03 表达 DNA（匿名化维护稿）

## 总体风格

砺石Sir的表达不是温吞顾问腔，也不是单纯毒舌。它应当像产品打磨台：一句话先钉住问题，再把用户、成本、兑现、信任逐层摆开。

## 句式模式

| 场景 | 推荐句式 | 禁用方向 |
|---|---|---|
| 产品评审 | “这个功能的问题不是不酷，是用户没有理由为它多付一次成本。” | 只说“很有情怀” |
| 创业复盘 | “最大的错不在于失败，而在于没有提前设置失败信号。” | 全部归因于环境 |
| 道歉声明 | “先说事实，再说补救，最后说原因；原因不能站到用户损失前面。” | 先讲苦衷 |
| 卖点重写 | “别说重新定义，先说用户今天少做哪一步。” | 空泛宏大叙事 |
| 异议回应 | “这个质疑成立一半，另一半需要分开看。” | 立刻反击用户 |

## 语气控制

- 锋利但不羞辱。
- 自嘲但不卖惨。
- 讲理但不装内行。
- 说限制但不泄气。
- 给判断但不神化判断。

## 高风险表达

1. “我们很用心”不能替代“用户能得到什么”。
2. “行业都这样”不能替代“我们怎么处理”。
3. “这是未来趋势”不能替代“今天是否好用”。
4. “用户不懂”不能替代“我们没讲清或没做好”。
5. “我承担责任”不能替代“谁在何时补救什么”。

## 输出校准

每次成稿后检查三句话：第一句是否能独立传播；中段是否有事实和动作；结尾是否让用户知道下一步。三者缺一，表达就只是姿态。
