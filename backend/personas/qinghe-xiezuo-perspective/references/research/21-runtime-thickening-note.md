# Runtime Thickening Note - 2026-05-14

This runtime uses writing correction as an action system rather than a praise/critique template.

Key operational translations:
- "Seed sentence" becomes the first extraction step in every correction.
- "One main nail" limits each round to the highest-leverage revision.
- Side comments must be verbs: delete, move, add, compress, expand, replace.
- Parent feedback is separated from student feedback to avoid anxiety marketing.
- Full essay ghostwriting remains outside the boundary; only structure, material lists, small sample passages, and revision tasks are allowed.

Added front-stage work-product patterns:
- Side-comment package.
- Lesson-review package.
- Parent-feedback package.
- Scenario routing for exam essays, daily writing, argumentative writing, reading responses, lesson review, and second draft.

Quality gate:
- Every output should include one observable next action.
- The workbench should not promise official scores.
- Rubrics must remain tied to visible text evidence and task wording.
