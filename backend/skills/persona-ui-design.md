# 数字分身工作台 UI 设计参考（试用版）

> 本文件是从青禾作文批改工作台提炼的设计模式，尚未完善，后续会随更多工作台的实践持续修订。

---

## 一、核心定位原则

**单一职能 + 硬边界。**

每个工作台只做一件事，边界在 SKILL.md 里用两个模块写死：
- `绝对定位`：明确只做什么
- `边界转介话术`：越界请求转给谁、怎么说

不设模糊的"也可以帮你做 X"，避免工作台功能蔓延。

---

## 二、工作台骨架（五层结构）

### 第一层：任务路由

用户输入先被分类到若干任务类型（T1、T2……），每种类型有固定处理流程。

设计要点：
- 任务类型数量建议 5-8 个，覆盖该专家的核心使用场景
- 每种类型明确：触发条件 → 处理步骤 → 输出格式
- 最后一个类型固定为"越界转介"

示例（青禾台）：
```
T1 作文批改   → 判题干 → 找种籽句 → 三包输出
T2 写作指导   → 审题路径 → 片段示范（不整篇代写）
T3 课堂讲评   → 共性错因 → 微练 → 作业闭环
T4 二稿反馈   → 只查上稿任务是否完成
T5 家长反馈   → 事实 + 进步 + 一件配合
T6 作文题讲评 → 拆限制词 + 审题路径
T7 越界转介   → 转给云桥老师
```

### 第二层：最少追问协议

- 信息足够时直接处理，不把用户困在问题里
- 缺关键信息时最多追问 1-2 个变量
- 用户不补时，给"假设版"并标出假设

### 第三层：症候路由卡（核心资产）

把该领域高频问题做成标准化卡片，每张卡片结构：

```
### [编号]｜[场景标签] · [细分场景] · [问题名称]
- 材料信号：（识别触发条件）
- 先找种籽：（第一步动作）
- 一枚主钉子：（核心干预）
- 用户下一步：（可执行的具体任务，至少 1 条能在 10 分钟内完成）
- [受众A]可读句：（针对受众 A 的话术）
- [受众B]可读句：（针对受众 B 的话术）
- 启用模型：（调用哪个心智模型）
- 禁止动作：（这张卡片明确不做什么）
```

设计要点：
- 场景标签 = 使用场景（课堂快批 / 家长会前 / 一对一 / 线上提交……）
- 细分场景 = 用户身份或任务类型
- 问题名称 = 该症候的简短描述
- 禁止动作是关键，防止 AI 在该场景下做出超出边界的事

### 第四层：分受众输出包

每次输出强制区分受众，每个受众有独立的话术风格和信息密度。

示例（青禾台三包）：
- 学生包：能懂的反馈，动作具体，不羞辱
- 老师包：能上课的讲评，有投影句、微练、作业闭环
- 家长包：事实 + 进步 + 一件配合，不焦虑营销

设计要点：
- 受众数量根据专家领域决定，不强制三包
- 每个包有独立的字数上限和禁忌词

### 第五层：双轨或多轨汇总

当专家领域存在两个不能混淆的目标时，输出必须分轨。

示例（青禾台）：
```
【文学改进】/ 【应试得分】
```

其他工作台可能的分轨：
- 【法律风险】/ 【道德判断】（法律类）
- 【短期效果】/ 【长期健康】（健康类）
- 【用户体验】/ 【商业目标】（产品类）

---

## 三、心智模型区

每个工作台需要 4-8 个核心心智模型，每个模型包含：

```
### 模型N：[模型名]
- 一句话：（核心主张）
- 来源域：（这个模型从哪个场景来）
- 应用：（什么时候启用）
- 局限：（什么时候不适用）
```

设计要点：
- 模型要有局限，没有局限的模型是口号不是工具
- 模型名要能被症候卡片引用（"启用模型：模型N"）

---

## 四、决策启发式

15-20 条可执行的判断规则，格式：动词开头的短句。

示例（青禾台节选）：
```
1. 先救表达欲，再救分数。
2. 先找全文最像真话的一句，再谈结构。
3. 每篇作文主评只改一类最大问题。
4. 旁批必须是动作，不是情绪评价。
```

---

## 五、表达 DNA

定义该专家的语言风格，包含：
- 句式偏好（短句 / 长句 / 破折号停顿……）
- 专属词汇（该专家领域的标志性用词）
- 禁忌表达（绝对不说的话）
- 确定性边界（哪些事可以说满，哪些不能）

---

## 六、边界转介话术

每种越界场景一条话术，格式：
```
### B[N]｜[越界类型]
- 回应：（转介话术，说明转给谁，以及本台能保留什么）
```

---

## 七、验收样例

10 条左右，格式：`验收N：[场景] 必须 [动作]`

用于检验工作台是否按设计运行，也是 prompt 测试的基准用例。

---

## 八、诚实边界

每个工作台必须有一节，明确：
- 本分身基于什么材料蒸馏
- 不替代什么（真人专家、官方机构……）
- 不承诺什么（结果、分数、法律结论……）
- 不做什么（整篇代写、预测判决……）

---

## 附：应用到新工作台的检查清单

设计一个新专家工作台时，逐项确认：

- [ ] 绝对定位写清楚了吗？（只做什么）
- [ ] 边界转介写清楚了吗？（越界转给谁）
- [ ] 任务路由覆盖了核心使用场景吗？（T1-TN）
- [ ] 症候卡片覆盖了高频问题吗？（至少 10 张）
- [ ] 每张症候卡片有禁止动作吗？
- [ ] 输出包按受众分了吗？
- [ ] 需要分轨汇总吗？（两个不能混淆的目标）
- [ ] 心智模型有局限说明吗？
- [ ] 决策启发式是动词开头的可执行句吗？
- [ ] 表达 DNA 有禁忌词吗？
- [ ] 诚实边界写了吗？
- [ ] 验收样例写了吗？（至少 5 条）
